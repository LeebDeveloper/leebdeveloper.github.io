Планирование NIF

Суть проблемы

  • Erlang считает редукции ("операции")
  • NIF — нативный код
  • Можно бампать счётчик редукций (enif_consume_timeslice)
  • Нельзя прерваться посередине
  • Долгий NIF — блокируем планировщик

Как работает NIF

  • Загружаем модуль
  • Загружаем .so
  • Получаем список функций
  • Патчим Erlang-код

Erlang

-module(niftest).
-export([init/0, hello/0]).
init() ->
      erlang:load_nif("./niftest", 0).
hello() ->
      "NIF library not loaded".

C

#include "erl_nif.h"

static ERL_NIF_TERM hello(ErlNifEnv* env, 
							int argc, const ERL_NIF_TERM argv[])
{
    return enif_make_string(env, "Hello world!", ERL_NIF_LATIN1);
}

static ErlNifFunc nif_funcs[] = { {"hello", 0, hello} };

ERL_NIF_INIT(niftest,nif_funcs,NULL,NULL,NULL,NULL)

Department d <- getDepartments(s) ;*

return sum;

Что общего?

Между ; , ;? и ;*

  • Не занимаются вычислениями
  • Не зависят от типов значений
  • Вообще-то, имеют одинаковый тип

Тип? Какой тип?

Тип >=

a -> (a -> b) -> b

m a -> (m a -> m b) -> m b

(>>=) :: m a -> (a -> m b) -> m b

Монада — это тройка

  • m — полиморфный тип с одним аргументом (Maybe a, IO a, etc.)
  • (>>=) :: m a -> (a -> m b) -> m b
  • return :: a -> m a

Так что с побочкой?

Любые побочные эффекты = явные действия над "миром"


main :: World -> World
main w0 =
    let (v1,w1) = action1 w0 in
    let (v2,w2) = action2 v1 w1 in
    let (v3,w3) = action3 v2 w2 in
    action4 v3 w3

Перепишем с помощью bind


main :: World -> World
main w0 =
    action1 >>= action2 >>= action3 >>= action4

Несколько трюков

Введем тип

data IO a = IO {unIO :: State# RealWorld -> (# State# RealWorld, a #)}

Разбавим все синтаксичесим сахаром: do-notation
